Overview

The Intrasonics Decoder Library provides decoding functionality and a simple API, to allow applications to decode Intrasonics-encoded audio and receive events based on the data within it.

These pages describe the Android Intrasonics Decoder Library, and explain how to use it to enable decoding within Android applications.

This guide assumes that the reader is an experienced Android developer, and there is therefore no explanation of the fundamentals of how to create and build an Android project within Eclipse or Android Studio, or how to run the resulting app on an Android device.

All descriptions and examples are based on Eclipse Kepler, which is freely available for all major hardware platforms from https://www.eclipse.org. The Android Developer Toolkit (ADT), which enables Android velopment within Eclipse, is available from http://developer.android.com/tools/help/adt.html.

The Android Decoder Library distribution consists of a native "shared object" (a .so file), a jar library, and documentation that can all be dropped into an existing Android project. Functionally, the Decoder Library comprises a core decoder and a set of engines that provide behaviours for specific use cases. The set of engines and range of codewords are licensed with an encrypted token provided by Intrasonics, and available from the Intrasonics Developer Portal.

If the token is configured to automatically update itself, the Decoder Library will periodically contact an Intrasonics server to check whether a newer replacement token is available. This check is based on the app identifier, which must be registered with the Intrasonics Developer Portal to enable automatic updates for that app. If a new token is available, the Decoder Library will download and install it.

To accompany the Decoder Library, a set of example apps are also available from the Intrasonics Developer Portal to show the basic functionality and an example use of the various engines, as well as pre-encoded content that works with the example apps.

Decoder Plugins (Advanced Developers Only)

The Decoder Library employs a plugin architecture to allow decoding functionality to be extended.

Plugin development is for advanced developers only; in general, the standard SDK is expected to provide all the functionality that might be required.
