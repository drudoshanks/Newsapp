Decoder Basics

This page describes the contents of the Android Decoder Library zip file, and provides further details on how the Decoder works. A general understanding of this will help when integrating the Decoder into a project.

Contents of the Library

The Android Decoder Library consists of the following files, arranged in a folder structure that's ready for importing directly into an Eclipse project (see Getting Started for how to do this):

File path Description
libs/IntrasonicsDecoder.jar This contains the Java classes that are required to access the functionality of the low-level shared objects, providing access to the Decoder and Engines, and allowing events to be received by the app.
libs/*/libAudioDecoder.so The shared objects that contain the Decoder and Engines are contained inside architecture-specific folders. There will normally be 4 folders, named armeabi, armeabi-v71, mips and x86. These will all be built into any app that uses the Decoder Library, allowing the app and Decoder to be run on any current Android hardware platform.
libs/docs This folder contains the API documentation for the Decoder classes. The properties file mentioned below allows Eclipse to use the API documentation in its code editor to provide Javadoc help, tooltips, and automatic code completion.
libs/IntrasonicsDecoder.jar.properties This properties file links the classes in the IntrasonicsDecoder.jar with the API documents in libs/docs.
res/drawable/*.png
res/anim/i_anim_*.xml		 This folder contains Intrasonics' standard set of Decoder confidence images, and predefined animations for the images. See the Decoder Confidence section for information on how to include and use these in a project.
