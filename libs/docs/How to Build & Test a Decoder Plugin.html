How to Build & Test a Decoder Plugin

Note: In order to use a Decoder Plugin in the PluginEngineExample, a token that explicitly allows the Plugin Engine is required (evaluation SDK tokens do not allow the Plugin Engine). Please contact Intrasonics for a Plugin Engine-compatible token.

The provided projects are expected to be used with Eclipse in a Unix environment. If the projects are to be developed under Windows, the developer should install Cygwin to get the appropriate UNIX tools;

  • Download and install the latest version of Eclipse from https://www.eclipse.org (on some platforms you may need to install Java SE as well)

  • Download and install the latest version of the Android Developer Tools plugin for Eclipse from http://developer.android.com/tools/help/adt.html

  • Download and install the latest version of the Android NDK from http://developer.android.com/tools/sdk/ndk/index.html

  • Create an environment variable NDK_HOME pointing to the install location of the NDK.

  • Add the NDK install location to your PATH environment variable.

  • Download the Decoder plugin SDK from http://portal.intrasonics.com.

  • Open a new Eclipse workspace and import the following projects:

    • <path-to-test-harness>/PluginEngineExample as an Android project

    • <path-to-test-harness>/TestPlugin as a C/C++ Makefile project (ensure that you choose the Android toolchain rather than the default option of None)

  • Build all (ensure that they build in the correct order);

    1. TestPlugin (ensure that the libTestPlugin.so has been copied to Example/libs/armeabi)
    2. Example

  • Ensure that a handset is connected and that Eclipse can see it through the DDMS perspective.

  • Right click on the Example project and select Run As -> Android Application.

It is expected that most plugins would be started from the Test Plugin sample project and modified to suit the developer's particular needs.
