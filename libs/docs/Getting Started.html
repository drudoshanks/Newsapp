Getting Started

To get started with the Intrasonics Decoder, you can either use this guide to add decoding to an existing project, or open one of the included example projects that's most relevant to your particular need of the Decoder:

Trigger Engine Example Project

The Trigger Engine is the simplest engine provided. It simply propagates codewords received from encoded audio to the engine listener.

Further details, and an example project that uses the Trigger Engine, can be found here

Event Engine Example Project

The Event Engine is slightly more complex than the Trigger Engine. It accurately synchronises with audio tracks that have been encoded with Event Engine codewords, and then generates events at exact times relative to the start of the encoded audio.

Further details, and an example project that uses the Event Engine, can be found here

Heartbeat Engine Example project

The third engine is the Heartbeat Engine. This interprets continuous codewords that are inserted in real-time into broadcast audio, and reports to the engine listener a periodic event detailing the absolute time within the audio, based on the date and time of the original broadcast.

Further details, and an example project that uses the Heartbeat Engine, can be found here

Adding the Intrasonics Decoder to an Existing Eclipse Project

To add the DecoderLibrary to an Eclipse project, and make decoding functionality available to that project, perform the following steps.

Note that screenshots below are from Eclipse running on a Mac, but the instructions will apply equally to other platforms.

Unzip the decoder library zip file IntrasonicsDecoder.zip to reveal the binary shared object, jar file and documentation within. These are unzipped into folder called libs, which corresponds to the libs folder in an Android project.

Inside the extracted libs folder will be the IntrasonicsDecoder.jar file, sub-folders containing shared objects for different architectures, and a docs folder containing embedded API documentation for Eclipse. The API documentation can also be found on-line here.

Drag the extracted libs folder onto your project in Eclipse, as shown below.

When prompted, select to copy files and folders into the project:

Finally, if libs already exists, you'll be asked if you want to overwrite it:

Select No, which will make Eclipse copy new files and folders into libs without deleting any existing files or folders.

Finally, close and re-open the project, to allow Eclipse to index the Decoder API documentation that has just been added to the project.

Intrasonics Decoder Permissions

The Intrasonics Decoder receives audio from the device’s microphone, so it must declare that it requires permission to do so in the app’s manifest. Make sure that AndroidManifest.xml for your project contains the following:

<uses-permission android:name="android.permission.RECORD_AUDIO" />
